Page Functionality — Step-by-step

Sprint 3 Final
H

Home Page

Entry point presenting features, mission, and quick links to Sign Up and Login.

  1. Hero with value proposition and CTA to Sign Up.
  2. Feature cards (Task manager, Group project, Dashboard).
  3. Footer with About & Contact links.
S

Sign Up Page

Creates new user accounts and validates input.

  1. User fills Full name, Email, Password (client validation).
  2. POST to signup.php → server validates & hashes password.
  3. On success: redirect to Login with success message; on error: show form errors.
L

Login Page

Secure access for registered users.

  1. User enters email + password and submits form.
  2. POST to login.php → server verifies with password_verify().
  3. If successful: start session and redirect to Dashboard; else show error.
D

Dashboard Page

Shows task summaries, upcoming deadlines and progress metrics.

  1. Load user's tasks via server call (or AJAX) on page load.
  2. Render summary cards: total tasks, completed, overdue, active projects.
  3. Quick actions: Add task, go to Task Manager, open group projects.
T

Task Management Page (To-Do)

Full CRUD for personal tasks with filters and sort.

  1. User clicks "Add Task" → form opens (title, desc, priority, due date).
  2. Form POST/AJAX → taskManager.php?action=create → model inserts with prepared statement.
  3. Tasks listed with edit/delete/complete actions; UI updates via JS or page refresh.
  4. Search, filter by priority/date, sorting options (planned).
G

Group Project Page

Manage projects, assign members and track team progress.

  1. Create project: project name, description, leader.
  2. Add members → stored in Project_Members.
  3. Create project tasks → assign to members (stored in Project_tasks).
  4. Progress tracking: tasks status, project completion % on Dashboard.
A

About / Contact / User Guide

Show team info, contact form, and a user guide page.

  1. Contact form POST → option to store messages or send email server-side.
  2. User Guide: step-by-step screenshots and explanations for each module.
  3. About: team member roles and contributions.
